# Welcome to the v2.2 'Correction Preview' Beta

Welcome to the frontier of surgical prompt engineering. You are among a select group of users invited to test the **Reasoning IDE** and the **\"Fix-It\"** capability.

The goal of this Beta is to move from *observing* reasoning loops to *correcting* them instantly.

---

## 🚀 Getting Started

### 1. Activate the Beta
The v2.2 features are currently behind a feature flag. To enable them in your environment, run the following command in your terminal:

```bash
tw beta activate v2.2-correction
```

### 2. Triggering a Loop
To see the tool in action, you can either run your own complex agents or use our built-in stress test:

```bash
tw sample-run --stress-test
```

### 3. Using the \"Fix-It\" Button
Once you have a trace open in the Reasoning IDE:
1. Look for the red **`[Reasoning Loop]`** highlight in the trace.
2. Click the **Magic Wand ($\text{\ud83e\ude84}$)** icon (the \"Fix-It\" button) next to the loop.
3. A modal will appear containing:
   - **The Rationale:** An analysis of *why* the agent is looping.
   - **The Suggested Fix:** A precise prompt adjustment to resolve the issue.

### 4. Applying the Fix
If the suggestion looks correct, you can apply it to your prompt. We recommend testing the fix in a separate branch or session to compare the results against the looping trace.

## 🧪 Feedback & Contribution

We are building the **Prompt Surgery Gallery**—a living library of the most interesting reasoning failures and their cures. Your contributions are vital.

### How to Report a Result
Every "Fix-It" suggestion has two feedback buttons:
- ✅ **It Worked:** Tells us the meta-prompt successfully identified and solved the loop.
- ❌ **Report Failure:** Tells us the suggestion was hallucinated or ineffective. **Please use this liberally; it is our most valuable data point.**

### Contributing to the Gallery
If you encounter a particularly complex loop that the "Fix-It" button solved (or if you found a manual fix that worked where the tool failed), we want to feature it in the Gallery.

**To submit a 'Surgery':**
1. Capture the **Before** (the looping trace).
2. Capture the **Fix** (the prompt change).
3. Capture the **After** (the successful trace).
4. Submit via the dashboard or email `beta-feedback@tinycompany.ai` with the subject `[Gallery Submission]`.

---

## 🛠 Troubleshooting & Support
- **Command not found?** Ensure you have updated to the latest version of the TraceWhisper CLI.
- **Fix-It button missing?** Ensure you have successfully run the activation command and that the trace has been flagged as a `[Reasoning Loop]`.
- **General Bugs:** Please report all technical issues via the `#beta-support` channel in the community Discord.
