# v2.2 Prompt Surgery Gallery Tracker

This document serves as the internal ledger for tracking user-submitted 'surgeries' from the v2.2 Beta. The goal is to curate a high-signal set of examples for the public launch.

## 📈 Submission Pipeline
1. **Intake:** Submissions arrive via `beta-feedback@tinycompany.ai` or the Beta Dashboard.
2. **Verification:** Marketing/Product verifies the "Before" (Loop) and "After" (Fix) traces.
3. **Categorization:** The surgery is tagged by failure pattern (e.g., "Tool-Output Denial", "Contradictory Constraint").
4. **Publication:** Verified surgeries are added to `marketing/v2-2-prompt-surgery-gallery.md`.

## 🏷️ Failure Pattern Taxonomy
To ensure the gallery is organized, we use the following tags:
- `[Infinite Search]`: Agent repeats the same search query without broadening terms.
- `[Constraint Clash]`: Agent struggles between two opposing prompt instructions.
- `[Tool Denial]`: Agent refuses to accept valid tool output and repeats the call.
- `[Logic Loop]`: Agent repeats a reasoning step without progressing to the next.
- `[Hallucinated Tool]`: Agent attempts to call a tool that doesn't exist or is misused.

## 🎯 Target KPIs for Gallery
- **Total Verified Surgeries:** 20
- **Coverage:** At least 3 examples for each primary failure pattern.
- **Diversity:** Examples across at least 5 different agent use-cases (e.g., Research, Coding, Data Analysis).
