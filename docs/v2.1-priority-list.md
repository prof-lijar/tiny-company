# v2.1 Priority List: Removing the Gates

**Goal:** Accelerate the "Aha! Moment" by removing administrative and technical friction identified during Cohort 1 onboarding.

## Top 5 Priorities for v2.1

### 1. Click-Wrap Agreement (P0 - Critical)
*   **Problem:** The "Legal Wall." Manual Beta Testing Agreement signatures create a high-friction entry point.
*   **Solution:** Replace manual signatures with a click-wrap agreement integrated directly into the sign-up flow and the `tw login` CLI command.
*   **Success Criteria:** Zero manual legal steps between "Interest" and "API Key Acquisition."

### 2. SDK Silent Failure Prevention (P0 - Critical)
*   **Problem:** Integration Gap. Users may successfully initialize the SDK but fail to capture traces without knowing why.
*   **Solution:** Implement robust validation in `tracewhisper.init()` with explicit, actionable error messages (e.g., "Invalid API Key," "Network Timeout," "Log Path Unreachable").
*   **Success Criteria:** 100% of initialization failures result in a clear error message rather than silence.

### 3. Framework-Specific Recipes (P1 - High)
*   **Problem:** Integration complexity. Even with a simple SDK, developers prefer "copy-paste" solutions for their specific stack.
*   **Solution:** Create high-quality, framework-specific integration snippets for:
    *   LangChain
    *   CrewAI
    *   AutoGen
*   **Success Criteria:** Integration time (from SDK install to first trace) reduced to < 1 minute for these frameworks.

### 4. Enhanced First-Run CLI Guidance (P1 - High)
*   **Problem:** The gap between "Trace Captured" and "Narrative Seen." Users may not know to run `tw live` immediately.
*   **Solution:** Implement a "First-Time User" (FTU) experience in the CLI. After the first successful trace is detected, the CLI should explicitly prompt the user to open a new terminal and run `tw live` to see the magic.
*   **Success Criteria:** Increased percentage of users who reach the `tw live` state within their first session.

### 5. "Debug-First" Narrative Tuning (P2 - Medium)
*   **Problem:** Positioning misalignment. Users want a "Reasoning IDE," not just a summary.
*   **Solution:** Refine the LLM prompts used for narrative generation to aggressively highlight:
    *   **Reasoning Loops:** Where the agent repeats itself.
    *   **Contradictions:** Where the agent ignores a previous observation.
    *   **Pivots:** The exact moment the strategy changed.
*   **Success Criteria:** Qualitative feedback indicating the tool feels like a "debugging environment" rather than a "reporting tool."

---

## Success Metrics for v2.1
*   **Conversion Rate:** Increase Invite $\rightarrow$ Onboarded rate from 0% to $> 40\%$.
*   **TTFN (Time-to-First-Narrative):** Reduce average TTFN to $< 5$ minutes from the moment of sign-up.
