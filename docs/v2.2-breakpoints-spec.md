# Product Specification: v2.2 Interactive Break-points (The "Intervention")

## 1. Overview
Interactive Break-points allow developers to move from passive observation to active intervention. Instead of reviewing a failure after the fact, a developer can set triggers that pause the agent's execution in real-time. This enables "live-debugging" of the agent's reasoning, allowing the developer to inject a "nudge" or correction into the context and observe how the agent pivots its strategy.

## 2. User Flow

### 2.1 Setting a Break-point
1. **Configuration**: The developer defines a break-point trigger in their code or via the CLI.
    - *Example (Code)*: `tracewhisper.set_breakpoint(on='contradiction')`
    - *Example (CLI)*: `tw set-breakpoint --trigger "reasoning_loop"`
2. **Execution**: The agent starts its task.
3. **Trigger**: The TraceWhisper engine detects the specified event (e.g., a `[Contradiction]`) in the live stream.
4. **Pause**: The agent's execution is immediately suspended. The system sends a signal to the agent framework to halt.

### 2.2 The Intervention
1. **Notification**: The developer receives a notification (CLI alert or Dashboard popup): *"Break-point hit: [Contradiction] detected at Step 15."*
2. **Inspection**: The developer reviews the current state: the last 3 thoughts, the current action, and the contradiction.
3. **Injection**: The developer writes a "Nudge" (a corrective instruction).
    - *Example*: *"You are contradicting your previous finding that the API is down. Re-verify the status code before trying again."*
4. **Resume**: The developer clicks **"Resume with Nudge"**.
5. **Observation**: The nudge is injected into the agent's context as a "System Intervention," and the agent continues execution.

## 3. Input/Output

### 3.1 Break-point Triggers (Inputs)
The system must support the following trigger types:
- **Event-Based**: 
    - `on_contradiction`: Pauses when the agent says X and then Y (where Y $\neg$ X).
    - `on_loop`: Pauses when the same thought/action sequence repeats $N$ times.
    - `on_tool_failure`: Pauses when a critical tool returns an error.
- **Step-Based**:
    - `at_step(n)`: Pauses exactly at step $N$.
- **Pattern-Based**:
    - `on_text("regex")`: Pauses when a specific keyword or pattern appears in the agent's thought process.

### 3.2 Intervention Payload (Output/Injection)
The intervention consists of:
- **The Nudge**: A string of text to be appended to the agent's current prompt/context.
- **Context Shift**: An optional command to "Clear Memory" or "Reset to Step X" before resuming.

## 4. Acceptance Criteria

An Interactive Break-point is considered successful if:
1. **Low Latency Pause**: The agent halts execution within < 1 second of the trigger event being detected.
2. **State Preservation**: The agent's full state (memory, variables, history) is preserved during the pause.
3. **Effective Injection**: The "Nudge" is successfully integrated into the agent's context, and the subsequent agent action reflects the nudge.
4. **Resume Capability**: The agent can resume execution from the exact point of the pause without needing a full restart.

**Success Metric:**
- **MTTR (Mean Time to Resolve)**: Reduction in time spent on iterative prompt testing by allowing "live" corrections.

## 5. Integration

### 5.1 SDK Integration (Critical)
- **Pause Mechanism**: Requires a tight integration with the agent framework (e.g., using a blocking call or a signal) to ensure the agent actually stops.
- **Injection API**: A method `tw.inject_nudge(text)` that modifies the prompt buffer.

### 5.2 CLI Integration
- **Interactive Prompt**: A blocking CLI prompt: `[BREAKPOINT] Contradiction detected. Enter nudge or press Enter to resume: > `
- **Status Display**: A clear indicator that the agent is currently `PAUSED`.

### 5.3 UI Integration (Dashboard)
- **Live Pause State**: The dashboard changes color/state to "PAUSED" with a flashing alert.
- **Nudge Input Box**: A text area for entering the correction, with a "Resume" button.
- **Visual Timeline**: A marker on the trace timeline indicating where the intervention occurred.
