# Guide: The v2.2.2 Verification Loop

## Overview
The Verification Loop is an automated workflow designed to resolve reasoning failures (like infinite loops) without introducing regressions in other parts of the agent's behavior.

## How it Works
The loop follows a strict **Snapshot $\\rightarrow$ Try $\\rightarrow$ Verify $\\rightarrow$ Rollback** cycle.

### 1. The Workflow
1. **Trigger**: A reasoning failure is detected.
2. **Suggestion**: A `PromptFix` is proposed.
3. **Execution**:
    - **Snapshot**: The current system prompt is backed up.
    - **Apply**: The `PromptFix` is integrated into the system prompt.
    - **Verify (Local)**: The triggering input is re-run. The system checks if the loop is gone and if the output matches the `GoldStandard`.
    - **Verify (Global)**: The `VerificationSet` (benchmarks) is re-run to ensure no regressions.
4. **Verdict**:
    - **Success**: The fix is kept.
    - **Failure**: The fix is rejected.
    - **Regression**: The fix is flagged as unstable; the user is prompted to rollback.

### 2. Technical Implementation
The core logic is managed by `src.verification_loop.VerificationLoopManager`.

#### Key Methods:
- `apply_and_verify(...)`: Orchestrates the entire loop. Returns a `VerificationResult`.
- `rollback()`: Restores the system prompt to the last known good snapshot.

#### Telemetry Events:
The system emits the following events for UI synchronization:
- `ev_verify_start`: Loop initiated.
- `ev_verify_success`: Fix resolved the issue and passed benchmarks.
- `ev_verify_fail`: Fix failed to resolve the issue.
- `ev_verify_rollback`: System prompt restored to snapshot.

## UI Integration Mapping
Based on the Design Handoff, the UI should map states as follows:

| Event/Result | UI State | Color | Action |
| :--- | :--- | :--- | :--- |
| `ev_verify_start` | **Verifying...** | Signal Blue (`#38BDF8`) | Show checklist of benchmarks |
| `VerificationStatus.SUCCESS` | **Fixed & Stable!** | Insight Green (`#4ADE80`) | Confirm update |
| `VerificationStatus.FAILURE` | **Unresolved** | Obsidian Red (`#EF4444`) | Try another fix |
| `VerificationStatus.REGRESSION` | **Fixed but Unstable** | Alert Amber (`#FBBF24`) | Trigger `rollback()` |
