# v2.2 Power User Identification Criteria

To maximize the momentum of the Beta, we need to identify "Power Users"—those who are extracting the most value from v2.2—to conduct deep-dive interviews and generate more case studies.

## 📊 Telemetry Indicators
We will identify Power Users based on the following event patterns in the telemetry pipeline:

### 1. The "Aha!" Moment (High Velocity)
- **Indicator:** High frequency of `ev_fix_trig` (Fix-It triggered) followed by a successful run within a short window.
- **Meaning:** The user is actively using the "Fix-It" suggestions to solve real bugs.

### 2. The "Deep Diver" (High Engagement)
- **Indicator:** High number of `ev_trace_view` events on complex traces (>30 steps) combined with multiple `ev_narrative_read` events.
- **Meaning:** The user is relying on the Narrative to make sense of complex agent behavior.

### 3. The "Optimizer" (Iterative Improvement)
- **Indicator:** A pattern of `ev_fix_trig` -> `ev_apply_fix` -> `ev_fix_trig` (refinement loop).
- **Meaning:** The user is using the tool to iteratively polish their prompts to a high standard.

## 🎯 Target Profile for Interviews
We are looking for users who meet at least two of the above criteria AND:
- Are working on "Agentic" workflows (multi-step, tool-using).
- Have a high "Correction Rate" (ratio of fixes applied to total runs).

## 🛠️ Action Plan for Product/Eng
1. **Filter:** Once the data lag window closes, run a query for users with `ev_fix_trig` > 5 per day.
2. **Segment:** Group these users by their use case (Research, Coding, Data Extraction).
3. **Outreach:** Marketing will then reach out for 15-minute "Value Validation" calls.
