# User Guide: TraceWhisper

Welcome to **TraceWhisper**, the observability tool that turns chaotic AI agent logs into clear, human-readable narratives. This guide will help you get started with the CLI and understand how to interpret your Execution Reports.

## 1. Installation

*(Note: Installation steps are currently based on the local development environment)*
Ensure you have the project dependencies installed via `uv` or `pip`:
```bash
pip install -r requirements.txt
# or
uv sync
```

## 2. Basic Usage

The primary command for TraceWhisper is `parse`. This command takes a raw log file and generates a narrative report.

### Simple Parse
To generate a report from a log file using default settings:
```bash
python src/main.py parse path/to/your_log.json
```

### Specifying Output File
To save the result to a specific Markdown file:
```bash
python src/main.py parse path/to/your_log.json --output report.md
```

### Adjusting Verbosity
You can control how detailed the narrative is:
- `--verbosity concise`: Focuses only on Key Decision Points (KDPs) and the final result.
- `--verbosity detailed`: (Default) Provides a comprehensive story of the agent's journey.

## 3. Understanding the Execution Report

Every report generated by TraceWhisper follows a structured format:

### 3.1 Executive Summary
At a glance, you'll see:
- **Status:** Whether the agent succeeded or failed.
- **Goal:** The original objective the agent was pursuing.
- **Duration:** Total time taken for the execution.

### 3.2 The Journey
This is the core of TraceWhisper. Instead of a list of events, you get a narrative. 
- **Pivots:** Look for phrases like *"However, after encountering X, the agent decided to Y"*—these indicate critical shifts in strategy.
- **Reasoning:** The narrative connects the agent's internal thoughts to its external actions.

### 3.3 Tool Usage Table
A tabular summary of every tool called, including:
- Tool Name
- Input Arguments
- Outcome (Success/Error)

### 3.4 Failure Analysis
If the agent failed, this section is the most important. It identifies the **"Breaking Point"**—the exact moment the agent's reasoning diverged from the path to success or where a tool returned a fatal error.

## 4. Preparing Your Logs

TraceWhisper works best with logs that follow the **TraceWhisper Reference Format**. If you are using a different framework (e.g., LangGraph, CrewAI), you may need to map your logs to the following schema:
- `trace_id`: Unique identifier for the run.
- `events`: A list of `thought`, `action`, `observation`, and `final_response` objects.

For a detailed specification, see `docs/log-format-spec.md`.

## 5. Troubleshooting

- **File Not Found:** Ensure the path to your JSON log is correct and relative to the root directory.
- **Invalid JSON:** Ensure your log file is valid JSON. TraceWhisper will alert you if the parsing fails.
- **API Errors:** TraceWhisper requires an LLM for synthesis. Ensure your `OPENAI_API_KEY` or equivalent is set in your environment variables.
