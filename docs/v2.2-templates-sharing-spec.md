# Product Specification: v2.2 Reasoning Template Library & Team Logic Sharing

## 1. Overview
The Reasoning Template Library and Team Logic Sharing features move TraceWhisper from an individual developer tool to an organizational knowledge base. Instead of every AI Engineer having to rediscover the same prompt fixes for common reasoning failures (e.g., "The Infinite Loop"), the company can curate a library of "Proven Fixes" and share audit reports across the team.

## 2. Feature 1: Reasoning Template Library

### 2.1 Purpose
A curated database of common reasoning failure patterns and their corresponding prompt solutions.

### 2.2 User Flow
1. **Discovery**: A developer encounters a `[Reasoning Loop]` in their trace.
2. **Lookup**: The system automatically suggests: *"This looks like 'The Infinite Loop' pattern. View proven fix from Library."*
3. **Application**: The developer opens the template, reads the explanation of why this happens, and copies the suggested prompt snippet.
4. **Contribution**: After finding a new fix for a unique bug, the developer can "Submit to Library," providing the failing trace and the successful fix.

### 2.3 Template Structure
Each template in the library must contain:
- **Pattern Name**: (e.g., "The Recursive Search Trap")
- **Symptom**: Description of the behavior (e.g., "Agent calls the same tool with the same arguments 3+ times").
- **Root Cause**: Why LLMs typically do this (e.g., "Lack of explicit exit criteria in the system prompt").
- **Proven Fix**: The exact prompt snippet to add.
- **Example Trace**: A link to a real trace demonstrating the failure and the fix.

## 3. Feature 2: Team Logic Sharing

### 3.1 Purpose
Allow AI Engineers to collaborate on fixing complex reasoning chains without sharing raw logs or granting full system access.

### 3.2 User Flow
1. **Generation**: The developer generates a "Logic Audit Report" for a specific trace.
2. **Sharing**: The system creates a permanent, read-only URL (e.g., `tw.share/audit/trace_abc123`).
3. **Collaboration**: A teammate opens the URL and sees the narrative, the detected errors, and the "Fix-It" suggestions.
4. **Annotation**: Teammates can add comments to specific steps in the trace: *"I think the failure here is actually due to the API returning a 404, not a prompt error."*

### 3.3 Security & Permissions
- **Privacy Masking**: Before sharing, the system must automatically mask PII (emails, keys, passwords) in the narrative and logs.
- **Access Control**: URLs can be set to "Public," "Organization Only," or "Password Protected."

## 4. Acceptance Criteria

### 4.1 Template Library
- User can search for a failure pattern by keyword or symptom.
- User can apply a template fix to their current prompt in < 3 clicks.
- The library supports versioning (e.g., "Fix for GPT-4o" vs "Fix for Claude 3.5").

### 4.2 Team Sharing
- Shareable URLs render a full TraceWhisper narrative without requiring the viewer to have the local log file.
- PII masking correctly identifies and replaces sensitive strings with `[MASKED]`.
- Annotations are persisted and visible to all users with access to the URL.

## 5. Integration

### 5.1 Backend
- **Database**: A centralized store for templates and shared trace metadata.
- **URL Shortener/Router**: A service to handle the `tw.share` domain and route to the correct trace.

### 5.2 UI Integration
- **Library Browser**: A searchable gallery of reasoning patterns.
- **Share Button**: A "Generate Share Link" button in the Trace Viewer.
- **Comment Thread**: A side-panel for annotations on shared traces.
