# Product Specification: v2.2 Reasoning Efficiency Metric

## 1. Overview
The "Efficiency Score" is the primary quantitative KPI for TraceWhisper v2.2. It moves prompt engineering from "vibes-based" evaluation (looking at the final answer) to "process-based" evaluation (measuring the cognitive path). 

This document defines the rigorous mathematical formula for the Efficiency Score to ensure consistency across the `tw compare` tool and CI/CD guardrails.

## 2. The Efficiency Score Formula

The Efficiency Score ($\text{ES}$) is a percentage representing how close an agent's reasoning path is to the "Ideal Path."

$$\text{ES} = \left( \frac{\text{Minimum Possible Steps}}{\text{Actual Productive Steps}} \right) \times 100\%$$

### 2.1 Definitions
- **Actual Productive Steps**: The total number of steps taken to reach the correct goal, *excluding* steps identified as "Waste" (see Section 3).
- **Minimum Possible Steps**: The theoretical minimum number of steps required to solve the task, based on a "Gold Standard" baseline.

### 2.2 The "Waste" Deduction
Before calculating the ES, we must identify and isolate "Reasoning Waste." Waste does not count toward the "Productive Steps" but is tracked separately as a penalty.

**Waste Categories:**
1. **Reasoning Loops**: Repeated thoughts or tool calls with identical arguments.
2. **Contradictions**: Steps that explicitly negate a previous finding.
3. **Redundant Tool Calls**: Calling a tool to retrieve information already present in the context.
4. **Hallucination Spirals**: Attempting to use tools or APIs that do not exist.

$$\text{Total Steps} = \text{Productive Steps} + \text{Waste Steps}$$

## 3. Determining the "Minimum Possible Steps" (The Baseline)

Since "minimum steps" varies by task, TraceWhisper supports three baseline modes:

### 3.1 Expert-Defined Baseline (The Gold Standard)
For a specific gold-standard dataset, a human expert defines the optimal path.
- *Example*: "To solve X, the agent must: 1. Search Y, 2. Filter Z, 3. Format Output." $\rightarrow \text{Min Steps} = 3$.

### 3.2 Best-of-N Baseline (Empirical)
The system looks at the last 100 successful traces for the same task and takes the minimum number of productive steps found among them.
- $\text{Min Steps} = \min(\text{Productive Steps}_{1 \dots 100})$

### 3.3 Theoretical Lower Bound (Heuristic)
For simple tasks, the system calculates the minimum number of API calls required to satisfy the request based on the API documentation.

## 4. The "Waste Ratio" (Penalty Metric)

While the Efficiency Score measures *directness*, the Waste Ratio measures *instability*.

$$\text{Waste Ratio} = \frac{\text{Waste Steps}}{\text{Total Steps}}$$

A trace can have a high Efficiency Score (it eventually found the shortest path) but a high Waste Ratio (it looped 10 times before finding it). Both are reported in the `tw compare` output.

## 5. Example Calculation

**Scenario**: Task is to find a user's email.
- **Gold Standard Path**: 1. Search User $\rightarrow$ 2. Extract Email. ($\text{Min Steps} = 2$)
- **Actual Trace**:
    1. Search User (Productive)
    2. Search User (Waste: Loop)
    3. Search User (Waste: Loop)
    4. Search User (Productive - finally gets result)
    5. Extract Email (Productive)
- **Metrics**:
    - Total Steps: 5
    - Waste Steps: 2
    - Productive Steps: 3
- **Calculations**:
    - $\text{ES} = (2 / 3) \times 100\% = 66.6\%$
    - $\text{Waste Ratio} = 2 / 5 = 40\%$

## 6. Acceptance Criteria for Implementation
- [ ] `EfficiencyCalculator` module implemented in `src/`.
