# UI/UX Specification: v2.4 Enterprise Scale-Up

## 1. Design Philosophy: From Tool to Platform
v2.4 transforms TraceWhisper from a personal productivity tool into an **Enterprise Governance Platform**. The UI shifts from a "forensic" focus (what happened to this trace?) to a "strategic" focus (how is our organization's intelligence performing?).

### Enterprise UX Principles
- **Governance at a Glance:** High-level dashboards that aggregate data across teams.
- **Frictionless Promotion:** A clear path for promoting a "Private Fix" to a "Team Pattern" and finally to an "Enterprise Golden Path."
- **Audit-First Design:** Every change is tracked; the "Who, When, and Why" is always visible.
- **Integration-Native:** The UI should feel like a natural extension of the existing SDLC (e.g., integrated into Git PRs).

---

## 2. Enterprise User Flows

### 2.1 The Promotion Pipeline (Pattern Lifecycle)
**Goal:** Move a successful reasoning fix from an individual's experiment to a company standard.
1. **Discovery:** Engineer finds a fix using "The Fixer" in a private trace.
2. **Team Share:** Engineer marks the fix as "Proposed for Team Vault."
3. **Peer Review:** Team Lead receives a notification, reviews the "Reasoning Diff," and approves it.
4. **Enterprise Promotion:** Once a pattern is proven across multiple teams, the Org Admin promotes it to the "Global Golden Path."

### 2.2 Reasoning Peer Review (Git Integration)
**Goal:** Make reasoning a first-class citizen in the PR process.
1. **PR Trigger:** A developer submits a PR changing a system prompt.
2. **Automatic Trace-Diff:** The CI system generates a side-by-side "Reasoning Diff" based on a standard test suite.
3. **Reviewer Analysis:** The reviewer sees:
   - **Left:** The cognitive path of the old prompt.
   - **Right:** The cognitive path of the new prompt.
   - **Highlight:** The specific step where the reasoning improved (in **Insight Green**).
4. **Approval:** Reviewer approves the prompt change based on the *reasoning evidence*, not just the output.

### 2.3 Governance & Compliance Monitoring
**Goal:** Ensure all production agents adhere to the "Golden Path."
1. **Dashboard Entry:** Org Admin opens the Governance Dashboard.
2. **Compliance Heatmap:** A grid of agents showing compliance percentages.
3. **Drill-Down:** Admin clicks a "Non-Compliant" agent to see exactly where the reasoning deviated from the Golden Path.
4. **Remediation:** Admin triggers a "Force Alignment" request to the team lead.

## 4. Component Specifications

### 4.1 Enterprise UI Components
| Component | Visual Style | Interaction |
| :--- | :--- | :--- |
| **Golden Path Badge** | Gold border, Indigo text | Hover to see certification date and auditor |
| **Compliance Gauge** | Linear progress bar (Indigo/Slate) | Click to open deviation report |
| **Reasoning Diff Column** | Split view, Monospace font | Sync-scroll between Before/After |
| **Vault Breadcrumb** | `Org > Dept > Team` (Platinum separators) | Click any segment to jump up the hierarchy |

### 4.2 Enterprise Interaction Shortcuts
- `Ctrl+Shift+P`: Promote current pattern to Team Vault.
- `Ctrl+Shift+G`: Open Global Governance Dashboard.
- `Ctrl+Shift+V`: Verify current trace against Golden Path.
