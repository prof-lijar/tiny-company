# v2.2 Common Reasoning Failure Patterns Library

This document serves as the initial seed data for the Reasoning Template Library. It identifies common "cognitive traps" that AI agents fall into and provides the proven prompt snippets to resolve them.

## 1. The Infinite Loop (The Recursive Search Trap)
**Symptom**: The agent calls the same tool with the same arguments 3 or more times in a row, despite the tool returning the same result.
**Root Cause**: The agent lacks a "termination condition" or "exit criteria" in its system prompt. It believes that repeating the action will eventually yield a different result.
**Proven Fix**: 
> "If you have called the same tool with the same arguments more than twice without receiving new information, stop and analyze why the result is not changing. Do not repeat the same call a third time; instead, pivot your strategy or report the limitation."
**Example Trace**: `Search(query="X")` $\rightarrow$ `Search(query="X")` $\rightarrow$ `Search(query="X")`.

## 2. The Direct Contradiction (The Flip-Flop)
**Symptom**: The agent asserts a fact in Step N, and then asserts the opposite fact in Step N+1 or N+2.
**Root Cause**: Lack of a "working memory" verification step. The agent is reacting to the most recent observation without cross-referencing its previous conclusions.
**Proven Fix**:
> "Before committing to a conclusion, explicitly cross-reference it with your previous thoughts in this trace. If you find a contradiction, pause and explain the discrepancy before proceeding."
**Example Trace**: Step 5: "The server is UP." $\rightarrow$ Step 6: "The server is DOWN."

## 3. The Tool-Output Blindness (The Ignoring Trap)
**Symptom**: The agent calls a tool, receives a clear error or specific value in the output, but then acts as if the tool call failed or returned something else.
**Root Cause**: The agent is "hallucinating" the output or following a pre-conceived plan too rigidly, ignoring the actual observation from the environment.
**Proven Fix**:
> "When a tool returns a result, you must explicitly summarize the key finding from that output in your next 'Thought' block before deciding on the next action. Do not assume the outcome of a tool call before it has returned."
**Example Trace**: Tool returns `{"status": 404, "error": "Not Found"}` $\rightarrow$ Agent Thought: "The tool successfully retrieved the data, now I will process it."

## 4. The Goal Drift (The Rabbit Hole)
**Symptom**: The agent starts by addressing the user's primary goal, but becomes distracted by a minor detail and spends the rest of the trace optimizing that detail while forgetting the primary goal.
**Root Cause**: Lack of "Goal Persistence" instructions.
**Proven Fix**:
> "At every 3rd step, explicitly state your current progress relative to the primary goal: [Primary Goal]. If you find yourself focusing on a sub-task for more than 3 steps, evaluate if this is essential to the primary goal or a distraction."
**Example Trace**: Goal: "Find the CEO's email." $\rightarrow$ Agent spends 10 steps researching the history of the company's logo.

## 5. The Hallucination Spiral (The Ghost Tool)
**Symptom**: The agent attempts to use a tool or API parameter that does not exist in its provided toolset, often because it "remembers" this tool from its base training data.
**Root Cause**: Over-reliance on internal weights over the provided system context.
**Proven Fix**:
> "You are strictly limited to the tools provided in the current session. Before calling a tool, verify its name and parameters against the provided tool definitions. Do not invent tools or parameters based on prior knowledge."
**Example Trace**: Agent calls `fetch_internal_docs()` when only `search_web()` and `read_file()` are available.

## Integration Guide for CTO
These patterns should be loaded into the SQLite `reasoning_templates` table:
- `pattern_name`: The title (e.g., "The Infinite Loop")
- `symptom_regex`: A pattern to detect the loop in the trace.
- `proven_fix`: The prompt snippet provided above.
- `category`: Loop / Contradiction / Hallucination / Drift.
