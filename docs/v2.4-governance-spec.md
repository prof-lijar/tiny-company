# Functional Specification: v2.4 Reasoning Governance & Golden Paths

## 1. Introduction
Reasoning Governance ensures that AI agents within an organization do not just reach the correct answer, but do so via a **predictable, safe, and auditable cognitive process**. 

The core of this governance is the **Golden Path**: a standardized "Standard Operating Procedure" (SOP) for reasoning that defines the mandatory cognitive milestones an agent must hit before delivering a result.

---

## 2. Golden Path Definition

### 2.1 Conceptual Model
A Golden Path is a directed sequence of **Reasoning Milestones**. While the exact wording of the agent's internal monologue may vary, the *intent* and *outcome* of each milestone must be detectable.

### 2.2 YAML Schema Specification
Golden Paths are stored in the Enterprise Vault as YAML files.

```yaml
id: "gp-enterprise-retrieval-01"
name: "Secure Enterprise Data Synthesis"
version: "1.2.0"
description: "Standard path for agents accessing internal knowledge bases to ensure safety and verification."

# Global constraints for this path
constraints:
  max_deviation_score: 0.2
  require_strict_ordering: true

milestones:
  - id: "safety_filter"
    name: "Input Safety Check"
    order: 1
    required: true
    validation:
      type: "pattern_match"
      criteria: "contains('safety', 'policy', 'compliance')"
      description: "Agent must explicitly check the input against corporate safety guidelines."

  - id: "query_optimization"
    name: "Contextual Query Expansion"
    order: 2
    required: true
    validation:
      type: "tool_call"
      tool_name: "search_optimizer"
      description: "Agent must use the search_optimizer tool to refine the user query."

  - id: "multi_source_retrieval"
    name: "Cross-Reference Retrieval"
    order: 3
    required: true
    validation:
      type: "min_tool_calls"
      count: 2
      tool_category: "knowledge_base"
      description: "Agent must retrieve data from at least two distinct internal sources."

  - id: "verification_loop"
    name: "Fact Verification"
    order: 4
    required: true
    validation:
      type: "reasoning_block"
      tag: "verification"
      description: "Agent must perform a dedicated verification step to resolve contradictions between sources."

  - id: "final_synthesis"
    name: "Governed Synthesis"
    order: 5
    required: true
    validation:
      type: "pattern_match"
      criteria: "contains('based on the verified sources')"
      description: "Final output must explicitly reference the verification process."
```

## 3. Compliance Scanning Logic

### 3.1 The `tw verify-compliance` Tool
The CLI tool analyzes a reasoning trace against a defined Golden Path to determine if the agent "stayed on the path."

**Input:** 
- `trace_id` or `trace_file`
- `golden_path_id`

**Process:**
1. **Linearization:** The tool parses the reasoning trace into a sequence of events (Thought $\rightarrow$ Tool Call $\rightarrow$ Observation $\rightarrow$ Thought).
2. **Milestone Mapping:** For each milestone in the Golden Path, the scanner searches the trace for a matching event that satisfies the `validation` criteria.
3. **Sequence Validation:**
    - **Existence Check:** Are all `required: true` milestones present?
    - **Order Check:** If `require_strict_ordering` is true, do the mapped milestones appear in the specified `order`?
4. **Deviation Scoring:**
    - **Gap:** A missing required milestone.
    - **Jump:** Skipping a milestone or performing them out of order.
    - **Noise:** Excessive irrelevant reasoning steps between milestones (calculated as a ratio of `total_steps` to `milestone_steps`).

### 3.2 Reporting Output
The tool returns a JSON report and a CLI summary:

- **Status:** `COMPLIANT` | `DEVIATED` | `NON_COMPLIANT`
- **Adherence Score:** $0.0$ to $1.0$
- **Findings:**
    - `Missing: [milestone_id]`
    - `Out-of-Order: [milestone_id] occurred before [milestone_id]`
    - `Excessive Noise: High cognitive overhead detected between Step 2 and 3.`

## 4. Compliance Dashboard Requirements

The Governance Dashboard provides the "Enterprise AI Lead" a bird's-eye view of agent behavior across the organization.

### 4.1 Key Performance Indicators (KPIs)
- **Path Adherence Rate (PAR):** The percentage of production traces that are marked `COMPLIANT` for their assigned Golden Path.
- **Mean Time to Deviation (MTTD):** The average milestone index where agents typically deviate (e.g., "Most agents fail at the Verification Loop").
- **Governance Coverage:** The ratio of deployed agents that have a Golden Path assigned vs. those operating without governance.
- **Violation Density:** Number of compliance failures per 1,000 requests.

### 4.2 Required Views
1. **Global Governance Heatmap:**
    - X-axis: Agents/Teams.
    - Y-axis: Golden Paths.
    - Cell Color: Adherence Rate (Green $\rightarrow$ Red).
2. **The "Divergence Sankey":**
    - A flow diagram showing where traces deviate from the Golden Path.
    - *Example:* "60% of traces pass Step 1 & 2, but 30% jump straight from Step 2 to Step 5, skipping Retrieval and Verification."
3. **Agent Health Leaderboard:**
    - List of agents sorted by lowest adherence score, allowing leads to identify which agents need prompt surgery or APO tuning.

## 5. Enterprise Use Case Example: "Customer Support Escalation"

**Scenario:** An agent handling high-priority customer complaints must follow a strict legal and operational protocol before escalating to a human.

**Golden Path: `gp-support-escalation-v1`**
1. **Sentiment Analysis (Required):** Detect urgency and frustration levels.
2. **Policy Lookup (Required):** Check the specific customer's SLA and contract terms.
3. **Attempted Resolution (Required):** Must provide at least one viable solution based on policy.
4. **Escalation Justification (Required):** Explicitly state *why* the automated resolution failed.
5. **Hand-off Package (Required):** Construct a summary for the human agent including the trace of attempted fixes.

**Compliance Failure Example:**
- Agent detects sentiment $\rightarrow$ Agent looks up policy $\rightarrow$ Agent immediately escalates.
- **Result:** `NON_COMPLIANT`. **Reason:** Missing `Attempted Resolution` and `Escalation Justification`.
