# Product Specification: v2.2 'Fix-It' Button

## 1. Overview
The 'Fix-It' button is the cornerstone of the v2.2 "Closed-Loop Debugger" strategy. It transforms TraceWhisper from a diagnostic tool (Detection) into a corrective tool (Correction). When the system detects a reasoning failure (e.g., a reasoning loop or a contradiction), the 'Fix-It' button allows the developer to generate an automated suggestion for modifying the agent's system prompt to prevent the failure from recurring.

## 2. User Flow

### 2.1 Triggering the Suggestion
1. **Detection**: The TraceWhisper engine identifies a `[Reasoning Loop]` or `[Contradiction]` in a trace.
2. **Notification**: In the UI/CLI, the specific segment of the trace is highlighted with a warning icon.
3. **Action**: The user clicks the **"Fix-It"** button associated with that specific failure event.
4. **Processing**: The system gathers the relevant context and sends it to the Meta-Prompt engine.
5. **Delivery**: The system presents a "Suggested Prompt Adjustment" side-by-side with the original prompt.
6. **Application**: The user can either copy the suggested text or click "Apply to Prompt" (if integrated with the agent's config).

### 2.2 CLI Flow
- User runs `tw analyze <trace_id>`.
- The CLI identifies a loop at step 14.
- CLI prompts: `Reasoning Loop detected at step 14. Would you like a suggested fix? (y/n)`.
- If `y`, the CLI displays the suggested system prompt modification.

## 3. Input/Output (Meta-Prompt Interface)

### 3.1 Input Context
To generate a high-quality fix, the Meta-Prompt needs the following context:
- **Current System Prompt**: The exact prompt that produced the failing trace.
- **The Failure Segment**: The specific sequence of thoughts/actions that constituted the loop or contradiction.
- **The Goal**: The original user request/objective the agent was trying to achieve.
- **The "Why" (Analysis)**: The internal reasoning TraceWhisper used to flag the event as a `[Reasoning Loop]` or `[Contradiction]`.
- **Execution History**: The 3-5 steps preceding the failure to provide behavioral context.

### 3.2 Expected Output
The Meta-Prompt must return a structured response:
- **Analysis**: A brief explanation of *why* the current prompt allowed the failure (e.g., "The prompt lacks a termination condition for recursive searches").
- **Suggested Modification**: A specific, additive or corrective instruction to be added to the system prompt.
- **Rationale**: Why this specific change is expected to break the loop or resolve the contradiction.
- **Confidence Score**: (Low/Medium/High) based on the clarity of the failure pattern.

## 4. Acceptance Criteria

A 'Fix-It' suggestion is considered "Good" if it meets the following:
1. **Directly Addresses the Root Cause**: The suggestion targets the logic gap identified in the analysis, not just a superficial symptom.
2. **Non-Regressive**: The suggested change does not introduce contradictory instructions into the existing prompt.
3. **Actionable**: The output is a clear instruction (e.g., "Always verify X before proceeding to Y") rather than a vague suggestion (e.g., "Be more careful").
4. **Measurable Improvement**: In a controlled test, applying the fix results in a successful trace for the same input where the previous prompt failed.

**Success Metric:**
- **Fix-It Acceptance Rate**: $> 25\%$ of suggestions are adopted by the user without significant manual editing.

## 5. Integration

### 5.1 UI Integration (Dashboard)
- **Visual Cue**: A magic-wand icon $\text{🪄}$ next to detected reasoning errors.
- **Comparison View**: A diff-style view showing `Current Prompt` $\rightarrow$ `Suggested Prompt`.
- **One-Click Copy**: Quick-copy button for the suggested snippet.

### 5.2 CLI Integration
- **Flag**: `tw analyze --suggest-fix <trace_id>`.
- **Output**: Formatted Markdown block showing the "Analysis" and "Proposed Change".

### 5.3 Technical Integration
- **Backend**: A new endpoint `/api/v2/suggest-fix` that orchestrates the context gathering and Meta-Prompt call.
- **Meta-Prompt**: A specialized prompt template stored in the system's prompt library, optimized for "Prompt Engineering as a Service".
