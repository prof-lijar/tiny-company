# v2.2 Cost-per-Correction Tracking Model

## 1. Overview
This document establishes the Cost of Goods Sold (COGS) model for the v2.2 "Fix-It" functionality. As TraceWhisper transitions from a diagnostic tool to a corrective IDE, the primary variable cost shifts from simple trace ingestion to the execution of the Meta-Prompt engine.

The "Correction Loop" is defined as the sequence: **Analysis $\rightarrow$ Fix $\rightarrow$ Validation**.

## 2. Token Consumption Model
We estimate the token cost for a single "Fix-It" request. We assume the use of a high-reasoning model (e.g., GPT-4o or equivalent) to ensure the quality of prompt suggestions.

### 2.1 The "Fix" Request (Meta-Prompt)
The Meta-Prompt requires significant context to generate an accurate fix.

| Component | Estimated Tokens (Avg) | Notes |
| :--- | :--- | :--- |
| **Input: System Prompt** | 1,000 | The current prompt being debugged |
| **Input: Failure Segment** | 1,500 | The specific trace segment where the loop occurred |
| **Input: Execution History** | 1,000 | 3-5 steps preceding the failure |
| **Input: Analysis & Goal** | 500 | The 'Why' and the original objective |
| **Input: Meta-Prompt Template** | 500 | System instructions for the Meta-Prompt engine |
| **Total Input Tokens** | **4,500** | |
| **Output: Analysis & Fix** | **800** | Analysis, Suggested Mod, Rationale, Confidence |
| **Total Output Tokens** | **800** | |

### 2.2 Unit Cost Calculation
Using current market rates for frontier models (approx. $5/1M input, $15/1M output):

- **Input Cost**: $4,500 \times (\$5 / 1,000,000) = \$0.0225$
- **Output Cost**: $800 \times (\$15 / 1,000,000) = \$0.0120$
- **Total Cost per Correction**: **$\approx \$0.0345$**

*Note: If "Analysis" is performed as a separate pre-step LLM call, we add an estimated $\$0.015$ per request, bringing the total to $\approx \$0.05$.*

## 3. Revenue vs. COGS Analysis
The "Fix-It" button is exclusively available to **Pro** and **Enterprise** tiers.

| Tier | Monthly Price | Cost per Correction | Corrections to Break-Even (Monthly) |
| :--- | :--- | :--- | :--- |
| **Pro** | $39 | $0.0345 | **~1,130** |
| **Enterprise** | $149 | $0.0345 | **~4,318** |

### 3.1 Interpretation
The marginal cost of a "Fix-It" suggestion is negligible compared to the subscription price. A Pro user would need to generate over 1,100 corrections per month before the feature itself becomes unprofitable. Given the nature of prompt engineering, it is highly unlikely a single user will exceed this threshold.

## 4. Break-Even & Margin Impact
- **Gross Margin**: The gross margin for the Correction Suite is estimated at **> 98%** per user, assuming average usage of 50-100 corrections per month.
- **Variable Cost Impact**: Even if token costs increase by 10x (e.g., moving to a significantly more expensive model), the break-even for a Pro user remains at $\approx 113$ corrections/month, which is still well within a safe margin.

## 5. Risk Mitigation
To prevent "API exhaustion" or malicious usage:
1. **Rate Limiting**: Implement a hard cap on "Fix-It" requests per user per month (e.g., 500 for Pro, Unlimited for Enterprise).
2. **Context Truncation**: Use sliding windows for execution history to keep input tokens below 10k.
3. **Model Routing**: Route simpler failures to cheaper models (GPT-4o-mini) and complex failures to frontier models.
