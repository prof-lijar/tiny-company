# Brand Identity Guide: TraceWhisper (v2 Final)

## 1. Brand Essence
TraceWhisper is the "Forensic Analyst" of the AI agent world. Our brand represents the transition from chaos (raw logs) to clarity (narrative).

**Core Values:**
- **Precision:** We don't guess; we pinpoint.
- **Clarity:** We strip away the noise to reveal the signal.
- **Trust:** We provide an immutable, transparent audit trail.

## 2. Visual Identity: "Precision-Secure"
The visual vibe is inspired by modern IDEs, cybersecurity dashboards, and high-end forensic tools. It is dark, focused, and high-contrast.

### 2.1 Color Palette
We use a "Dark-First" approach to reduce eye strain for developers and highlight critical information.

| Role | Color Name | Hex Code | Usage |
| :--- | :--- | :--- | :--- |
| **Background** | Deep Space | `#0B0E14` | Primary background, terminal base |
| **Surface** | Obsidian | `#161B22` | Cards, modals, sidebars |
| **Primary** | Signal Violet | `#8B5CF6` | Primary actions, active states, highlights |
| **Secondary** | Whisper White | `#E6EDF3` | Primary text, high-emphasis labels |
| **Muted** | Slate Grey | `#8B949E` | Secondary text, captions, disabled states |
| **Success** | Neural Green | `#2ECC71` | Positive outcomes, resolved issues |
| **Alert** | Warning Amber | `#F1C40F` | Divergence points, warnings |
| **Error** | Critical Red | `#E74C3C` | Failures, crashes, critical bugs |

### 2.2 Typography
Our typography choice balances readability with a "technical" feel.

- **UI Typography:** `Inter` (Sans-serif)
    - Used for headers, labels, and general interface text.
    - Properties: Clean, modern, highly legible.
- **Data Typography:** `JetBrains Mono` (Monospace)
    - Used for logs, narratives, code snippets, and CLI output.
    - Properties: Optimized for reading code and structured data.

## 3. Logo Concept
The logo should visually represent the "Noise-to-Signal" transition.
- **Concept:** A series of scattered, fragmented dots on the left that gradually align into a single, sharp, focused line or a "whisper" wave on the right.
- **Symbolism:** The transformation of chaotic telemetry into a clear narrative.

## 4. Tone of Voice
TraceWhisper speaks as a **Trusted Expert**.

- **Professional but Direct:** We avoid fluff. We provide the answer and the evidence.
- **Empowering:** We don't just tell the user what went wrong; we give them the tools to fix it.
- **Analytical:** Our language is grounded in evidence (e.g., "The trace diverges at step 14" instead of "The agent seems lost").

## 5. Visual Asset Specifications
- **Favicon:** A stylized "TW" in Signal Violet on a Deep Space background.
- **Social Media:** High-contrast imagery featuring dark-mode code snippets with Signal Violet highlights.
- **Icons:** Thin-line, geometric icons. Avoid rounded, "bubbly" styles.

## 6. Brand Governance & Usage
To maintain consistency, the following rules apply to all TraceWhisper visual communications:

### 6.1 Logo Usage
- **Clear Space:** Maintain a minimum clear space around the logo equal to the height of the 'T' in the logo.
- **Forbidden Uses:** 
    - Do not use the logo on light backgrounds without the Deep Space container.
    - Do not apply gradients or shadows to the logo.
    - Do not distort the aspect ratio.

### 6.2 Color Application
- **Primary Action:** Use Signal Violet for the primary call to action.
- **Semantic Colors:** Neural Green, Warning Amber, and Critical Red must only be used for their respective semantic meanings (Success, Warning, Error) to avoid confusing the user.

### 6.3 Consistency Check
Every new UI component must be validated against the "Signal-to-Noise" philosophy: *Does this element add signal, or does it add noise?*
