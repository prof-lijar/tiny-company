# UI/UX Specification: v2.2 'Reasoning IDE'

**Status:** Final
**Version:** 2.2
**Designer:** [Designer]
**Date:** 2026-05-20

## 1. Introduction & Design Philosophy
The v2.2 update transforms TraceWhisper from a **Passive Monitor** into an **Active Reasoning IDE**. The goal is to minimize the "Iteration Gap"—the time between detecting a reasoning failure and deploying a fix.

### Design Principles:
- **Contextual Intervention**: Tools and buttons appear exactly where the failure occurs, not in a separate menu.
- **Evidence-Based Optimization**: Changes are driven by quantitative data ($\Delta$ metrics) and forensic evidence (Divergence points).
- **Non-Destructive Debugging**: The UI allows users to "nudge" agents and compare paths without losing the original trace context.

## 2. The 'Fix-It' Workflow (Closed-Loop Debugging)

### 2.1 Trigger and Placement
The 'Fix-It' action is a contextual intervention triggered by backend detection of `[Reasoning Loop]` or `[Contradiction]`.

- **Visual Indicator**: A high-contrast "Magic Wand" icon ($\text{\ud83e\ude84}$) appears in the right margin of the Trace Narrative, immediately adjacent to the failing step.
- **Interaction**: Clicking the icon opens the **Correction Drawer**.

### 2.2 The Correction Drawer (Right Panel)
A slide-over panel that maintains the trace in view while providing the fix.

**Components:**
1. **Analysis Section**: Plain-English explanation of the failure (e.g., *"The agent is stuck in a loop because it is searching for a file that does not exist"*).
2. **Prompt Diff View**: A side-by-side comparison of the System Prompt.
   - **Left (Current)**: Highlights problematic instructions in red.
   - **Right (Suggested)**: Highlights new/modified instructions in green.
3. **Rationale**: A brief explanation of why the suggested change resolves the specific failure.
4. **Confidence Badge**: A color-coded badge (Low/Med/High) indicating the AI's confidence in the fix.
5. **Action Footer**: 
   - `[Copy Suggestion]` (Primary)
   - `[Apply to Prompt]` (Secondary - pushes to prompt management system)

## 3. Comparative Trace Analysis (A/B Testing)

### 3.1 Comparison Header ($\Delta$ View)
When two traces are compared, the top navigation transforms into a performance dashboard.

**Metric Cards**:
Each card displays: `Baseline` $\rightarrow$ `Variant` | `Delta %`
- **Steps**: $24 \rightarrow 18$ | $\downarrow 25\%$ (Green)
- **Tokens**: $12k \rightarrow 9k$ | $\downarrow 25\%$ (Green)
- **Loops**: $2 \rightarrow 0$ | $\downarrow 100\%$ (Green)
- **Efficiency**: $65\% \rightarrow 88\%$ | $\uparrow 23\%$ (Green)

### 3.2 Dual-Pane Narrative Viewer
- **Synchronized Scrolling**: Scrolling one pane scrolls the other.
- **Divergence Marker**: A bold red vertical line ($\text{--- | ---}$) marking the first index where `traceA.steps[i].action != traceB.steps[i].action`.
- **Visual Dimming**: Identical steps are dimmed (opacity 50%); differing steps are highlighted with a white background and bold text.
- **Verdict Badge**: A floating footer indicating the "Winner" based on the Efficiency Score.

+-----------------------------------------------------------------------+
| [ Steps: -25% v ] [ Tokens: -25% v ] [ Loops: -100% v ] [ Eff: +23% ^ ] |
+-----------------------------------------------------------------------+
|         TRACE A (Baseline)         |         TRACE B (Variant)        |
|------------------------------------|----------------------------------|
| Step 1: Init Agent... (dimmed)     | Step 1: Init Agent... (dimmed)    |

## 4. Interactive Break-points (The Intervention)

### 4.1 Global Pause State
When a break-point is hit (e.g., `on_contradiction`), the entire UI enters a **Paused State**.
- **Visual Cue**: The top header bar flashes a soft amber color with the text `[PAUSED] Break-point Hit: Contradiction Detected`.
- **Overlay**: A semi-transparent veil is placed over the Trace View to prevent accidental interaction with historical data.

### 4.2 Intervention Panel (The Nudge)
An overlay modal appears in the center of the screen.

**Components:**
1. **State Review**: Displays the last 3 thoughts and the current action that triggered the pause.
2. **The Nudge Input**: A large text area for the developer to write a corrective instruction.
   - *Placeholder*: "Enter a nudge to correct the agent's reasoning..."
3. **Execution Options**:
   - `[Resume with Nudge]`: Injects the text and continues execution.
   - `[Resume without Nudge]`: Continues execution as is.
   - `[Reset to Step X]`: Rewinds the agent to a previous state before resuming.

### 4.3 Intervention Timeline Marker
Once the agent resumes, the point of intervention is permanently marked in the trace as a `[System Intervention]`. It is styled differently (e.g., a purple background) to distinguish it from the agent's own thoughts.

## 5. Interaction Patterns & Accessibility

### 5.1 Keyboard Shortcuts
- `Cmd + Shift + F`: Trigger 'Fix-It' for the selected step.
- `Cmd + Shift + C`: Enter 'Comparison Mode'.
- `Esc`: Close any open drawer or modal.

### 5.2 Accessibility (WCAG AA)
- **Color Independence**: All $\Delta$ metrics use both color (Green/Red) and icons ($\uparrow/\downarrow$) to convey meaning.
- **Focus Management**: When the Intervention Panel opens, focus is trapped within the modal and set to the Nudge Input field.
- **Motion**: Synchronized scrolling can be toggled off in Settings for users with vestibular disorders.

## 6. Developer Implementation Notes
- **Divergence Logic**: The red line is placed at the first index $i$ where `traceA.steps[i].action != traceB.steps[i].action`.
- **Sync-Scroll**: Implement using a shared scroll-offset listener between narrative panes.
- **Diffing**: Use a standard diffing library for the Prompt Diff view to ensure clear `+` and `-` markings.
- **Pause Mechanism**: The UI must poll the backend for the `PAUSED` state or receive a WebSocket event to trigger the global pause overlay.
