# v2.2 Beta Telemetry Specification

**Date:** 2026-05-20
**Purpose:** To quantitatively measure the "Aha! Moment" and the effectiveness of the 'Fix-It' engine during the v2.2 Beta.

## 1. The "Aha! Moment" Definition
For TraceWhisper, the "Aha! Moment" is defined as:
> *A user identifies a reasoning failure, applies a 'Fix-It' suggestion, and observes a successful resolution in the subsequent trace without further manual prompt editing.*

## 2. Required Telemetry Events

To track this, the following events must be emitted by the SDK and CLI:

| Event ID | Event Name | Trigger Condition | Key Properties | Purpose |
| :--- | :--- | :--- | :--- | :--- |
| `ev_beta_act` | `beta_activation` | User successfully runs `tw beta activate` | `user_id`, `timestamp`, `version` | Track activation rate of the cohort. |
| `ev_fix_trig` | `fix_it_triggered` | User clicks/triggers the 'Fix-It' button | `trace_id`, `failure_type` (e.g. Loop), `suggestion_id` | Measure the demand for automated correction. |
| `ev_fix_app` | `fix_applied` | User confirms "Apply to Prompt" | `trace_id`, `suggestion_id`, `time_to_apply` | Measure the acceptance rate of suggestions. |
| `ev_corr_succ` | `correction_success` | Subsequent trace after `ev_fix_app` is marked as 'Success' | `original_trace_id`, `new_trace_id`, `time_delta` | **The Primary KPI**: Validates that the fix actually worked. |
| `ev_tmpl_hit` | `template_match` | System matches a trace to a known library template | `template_id`, `confidence_score` | Measure the "Template Hit Rate" (Deterministic vs Stochastic). |
| `ev_user_val` | `value_feedback` | User responds to "Did this save you time?" | `trace_id`, `score` (1-5), `comment` | Qualitative validation of effort saved. |

## 3. Success Metrics (KPIs)

The following metrics will be derived from the events above:

### 3.1 Activation Rate
$$\text{Activation Rate} = \frac{\text{Count}(\text{ev\_beta\_act})}{\text{Total Invitations Sent}}$$
*Target: > 66%*

### 3.2 Fix Acceptance Rate (FAR)
$$\text{FAR} = \frac{\text{Count}(\text{ev\_fix\_app})}{\text{Count}(\text{ev\_fix\_trig})}$$
*Target: > 50%*

### 3.3 Correction Efficacy (The "Aha!" Rate)
$$\text{Efficacy} = \frac{\text{Count}(\text{ev\_corr\_succ})}{\text{Count}(\text{ev\_fix\_app})}$$
*Target: > 30% (Initial Beta target)*

### 3.4 Template Hit Rate
$$\text{Hit Rate} = \frac{\text{Count}(\text{ev\_tmpl\_hit})}{\text{Total Traces Analyzed}}$$
*Target: > 20% (Indicates the library is capturing common failures)*

## 4. Implementation Notes for CTO
- Events should be buffered locally and flushed to the telemetry endpoint every 5 minutes or upon session close to avoid latency in the 'Fix-It' loop.
- `ev_corr_succ` requires the system to link the "fixed" trace back to the "original" trace via a `parent_trace_id` or `correction_id`.
