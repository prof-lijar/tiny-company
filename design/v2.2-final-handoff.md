# v2.2 Implementation Handoff: Reasoning IDE & Correction Loop

## 1. Overview
This document serves as the final design handoff for the v2.2 release. It consolidates the visual requirements for the "Closed-Loop Debugger" features: **Interactive Break-points** and the **'Fix-It' Suggestion Engine**.

The goal is to ensure a seamless transition from detection (observing a loop) to intervention (pausing/nudging) to correction (updating the prompt).

---

## 2. Visual Identity Core
All v2.2 components must adhere to the established brand guidelines.

### Color Palette
| Role | Color | Hex | Application |
| :--- | :--- | :--- | :--- |
| **Background** | Obsidian | `#0F172A` | Main IDE/Terminal background |
| **Primary Accent** | Signal Blue | `#38BDF8` | Standard reasoning flow, active states |

## 3. Feature Component Mapping

### 3.1 The 'Fix-It' Engine
**Goal:** Transform a detected error into a prompt improvement.

| Component | Visual Element | Implementation Detail |
| :--- | :--- | :--- |
| **Trigger** | Magic Wand Icon ($\text{\ud83e\ude84}$) | Placed adjacent to `[Reasoning Loop]` or `[Contradiction]` tags. |
| **Comparison View** | Side-by-Side Diff | Left: Current Prompt (Greyed out) $\rightarrow$ Right: Suggested Prompt (Highlighted). |
| **Highlighting** | Insight Green Background | Only the *changed/added* text in the suggested prompt is highlighted. |
| **Action** | Primary Button | "Apply to Prompt" (Signal Blue) or "Copy Snippet" (Ghost button). |

### 3.2 Interactive Break-points (The Intervention)
**Goal:** Live-debug by pausing execution and injecting nudges.

| Component | Visual Element | Implementation Detail |
| :--- | :--- | :--- |
| **State Indicator** | PAUSED Banner | A flashing Alert Amber bar at the top of the UI/CLI when a breakpoint is hit. |
| **Nudge Input** | Text Area | A focused input box with a prompt: *"Enter correction nudge..."* |
| **Timeline Marker** | Red/Amber Vertical Line | A distinct marker on the trace timeline indicating where the human intervened. |
| **Resume Action** | "Resume with Nudge" | High-contrast button to trigger the injection and restart the agent. |

## 5. CLI vs IDE Consistency
To maintain a unified experience, the CLI must mirror the IDE's logic:

1. **Colors:** Use `rich` library to implement `Alert Amber` for loops and `Insight Green` for fixes.
2. **Flow:** 
   - `tw analyze` $\rightarrow$ Detection $\rightarrow$ `(y/n) Suggest Fix?` $\rightarrow$ Diff output.
   - `tw live` $\rightarrow$ Breakpoint $\rightarrow$ Blocking input prompt $\rightarrow$ Resume.
3. **Terminology:** Use consistent labels: *"Breaking Point"*, *"Correction Bridge"*, and *"Nudge"*.
