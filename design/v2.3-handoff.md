# v2.3 UI/UX Engineering Handoff: The Intelligence Layer

## 1. Overview
The v2.3 update evolves TraceWhisper from a passive observability tool into a proactive intelligence layer. The focus is on **organizational knowledge (Vault)**, **efficiency (Pruning)**, and **reliability (CRI)**.

## 2. Feature-to-Component Mapping

### 2.1 Pattern Vault (Cross-Project Intelligence)
- **User Flow:** `The Fixer` $\rightarrow$ `Vault Query` $\rightarrow$ `Suggestion` $\rightarrow$ `Apply`.
- **Key Components:**
    - `Vault Insight Card`: Purple-accented card showing the source project and the proven fix.
    - `Vault-Fix Badge`: A specific label in the Prompt Diff viewer indicating a fix sourced from the Vault.
- **Wireframe Ref:** Section 5.4 (Pattern Vault Insight).

### 2.2 Cognitive Pruning (Efficiency)
- **User Flow:** `tw prune <id>` $\rightarrow$ `Bloat Analysis` $\rightarrow$ `Pruned Comparison` $\rightarrow$ `Apply`.
- **Key Components:**
    - `Cognitive Bloat Signal`: Fog Grey text with strikethrough in the narrative.
    - `Pruning Comparison Panel`: Side-by-side view of Original vs. Pruned trace.
    - `Efficiency Score Badge`: High-contrast metric (e.g., "64% Efficient").
- **Wireframe Ref:** Section 5.2 (Pruning Report).

### 2.3 Cognitive Reasoning Integrity (CRI)
- **User Flow:** `tw verify-all` $\rightarrow$ `Test Failure` $\rightarrow$ `Cognitive Diff` $\rightarrow$ `Divergence Point`.
- **Key Components:**
    - `Cognitive Path Step`: State-based components (`OK`, `Diverged`, `Bloat`).
    - `Cognitive Diff Viewer`: Vertical comparison of Expected vs. Actual paths.
    - `Divergence Marker`: High-visibility amber indicator at the point of deviation.
- **Wireframe Ref:** Section 5.1 (The Cognitive Diff).

### 2.4 IDE Integration (VS Code)
- **Interaction:** Bidirectional linking between Prompt files and Narrative traces.
- **Key Components:**
    - `IDE Prompt-Trace Bridge`: Logic to map trace segments to line numbers.
    - `Quick-Fix Lightbulb`: Gutter icon triggering the Fixer/Vault flow.
- **Wireframe Ref:** Section 5.3 (IDE Integration).

## 3. Visual Tokens & Design System
All implementations must adhere to the updated `design/component-spec.md`:
- **Vault Purple:** `#A855F7` (Knowledge/Intelligence)
- **Bloat Fog:** `#94A3B8` (Redundancy/Waste)
- **Alert Amber:** `#FBBF24` (Divergence/Loops)
- **Insight Green:** `#4ADE80` (Proven Fixes/Success)

## 4. Implementation Notes for CTO
- **CLI Implementation:** Use `rich.panel.Panel` for Vault Cards and `rich.table.Table` for the Pruning and Diff views.
- **IDE Implementation:** Focus on the `TextEditor` decoration API for the "Quick-Fix" lightbulbs and the side-panel for the Narrative Trace.
- **Data Mapping:** The "Divergence Point" in CRI is calculated by comparing the sequence of `ReasoningUnit` IDs between the Gold Standard and the Actual Trace.
