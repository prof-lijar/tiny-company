# Contribution Guidelines

Thank you for your interest in contributing to TraceWhisper! We welcome contributions from the community to improve the tool's narrative synthesis, log parsing, and organizational governance capabilities.

## How to Contribute

### 1. Reporting Bugs
- Please open an issue on the repository to report a bug.
- Include a detailed description of the problem and a sample log file (sanitized of any sensitive data) that reproduces the issue.
- For Enterprise features (multi-tenancy, RBAC), please specify if the issue occurs in a single-tenant or multi-tenant environment.

### 2. Suggesting Enhancements
- If you have an idea for a new feature or improvement, please open a feature request issue.
- Describe why the feature is important and how it would benefit users.
- If suggesting improvements to the reasoning governance layer (Golden Paths, vaults), describe the intended governance model.

### 3. Pull Requests
- **Fork the repository** and create your own branch.
- **Ensure your code follows the same style** as existing code.
- **Ensure all tests pass** before submitting your PR.
- **Add tests for new functionality** to ensure no regressions.
- **Submit a PR** with a clear description of the changes and why they are important.
- **Security Review:** Contributions to the Enterprise Core (multi-tenancy, RBAC, and vaults) are subject to a mandatory security review to ensure tenant isolation and data privacy.

### 4. Code of Conduct
By contributing to the community, you agree to abide by our Code of Conduct. Please see `legal/code-of-conduct.md` for more details.

### 5. Legal
All contributions to this project will be taken under the MIT License. By contributing to this project, you agree that your contributions will be licensed under its MIT License.
