# Contributing Guidelines

Thank you for your interest in contributing to the TOPIK Learning Assistant! We welcome contributions from developers, linguists, and Korean language experts.

## How to Contribute

### 1. Reporting Issues
If you find a bug or have a feature request, please open an issue in the repository. Please be as detailed as possible, providing steps to reproduce the bug or a clear use case for the feature.

### 2. Proposing Changes
For larger changes, we recommend opening an issue first to discuss the proposed changes with the maintainers.

### 3. Submitting a Pull Request
- Create a new branch for your feature or fix (e.g., `feature/your-feature-name` or `fix/your-fix-name`).
- Ensure your code follows the project's coding standards.
- Provide a clear and descriptive commit message.
- Open a Pull Request (PR) against the `master` branch.
- Include a description of the changes and reference any related issues (e.g., "Closes #123").

## Content Contributions
If you are contributing TOPIK-style practice questions:
- **Originality**: All content must be original. Do NOT copy actual TOPIK test questions.
- **Alignment**: Ensure questions align with the 2026 TOPIK reform specifications.
- **Formatting**: Follow the data structures defined in `product/src/lib/data/`.

## Code of Conduct
Please refer to our [Code of Conduct](./code-of-conduct.md) for guidelines on community behavior.
